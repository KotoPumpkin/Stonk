# 股票模拟交易系统设计方案

## 一、需求总结

### 1. 用户角色
- **管理员**：拥有最高权限，可管理股票品类、机器人账户（增删改、绑定策略与资金）、真人账户（增删改、调整资金与持仓）。
- **机器人账户**：预设三类交易策略（散户游资、正规机构、做空/做多组织），初始资金与策略挂钩；自动根据策略在当前步进单位决策买卖。
- **真人账户**：可手动买卖股票；可启用量化大模型代理（自动给出买卖建议或全权委托交易）；查看个人持仓和资金。

### 2. 时间步进模式
- **四种模式**，由管理员控制步进：
  - **超短线**：秒级步进，真人不可操作，仅可观看机器人操作或启用量化代理；图表仅显示折线图。
  - **较短线**：小时级步进，真人可操作。
  - **短线**：天级步进，真人可操作。
  - **中线**：月级步进，真人可操作；但机器人账户仍按天步进决策（即每月内机器人会多次交易）。
- **交易规则**：每个步进单位，账户可选择买入、卖出或不操作；买入卖出数量不限，无涨跌停、ST等限制，随时可交易。

### 3. 交易规则
- 股票可做多也可做空（允许负持仓）。
- 交易市价成交（简化）。

### 4. 图表与指标
- 提供分钟K线、小时K线、天K线、月K线。
- 超短线模式仅折线图；其他模式可叠加MACD、KDJ等指标。
- UI要求简洁大方，扁平化设计。

### 5. 联机与步进控制
- 系统需支持多真人账户同时联机操作。
- 步进控制器仅限管理员使用。
- **步进流程**：管理员点击下一步后，所有真人账户需确认完成操作，服务器才执行下一步并更新状态。
- **快进功能**：管理员可开启快进，此时强制所有真人账户不能手动操作，可选择不操作或由量化代理托管；快进期间服务器自动步进，直至管理员停止。

### 6. 其他
- 系统为Windows桌面端程序（客户端-服务器架构，服务器也带管理员界面）。
- 量化代理初期可用规则模拟，后期可接入真实模型。

## 二、简洁设计方案

### 1. 系统架构
采用 **C/S（客户端-服务器）** 架构：
- **服务器端（管理员操作端）**：包含业务核心（步进控制器、策略引擎、交易引擎、价格生成器）、WebSocket服务、数据库（SQLite）、管理员GUI（PySide6）。
- **客户端（真人用户端）**：提供登录、交易面板、图表、量化代理开关等功能，通过WebSocket与服务器通信。

### 2. 技术选型
- **编程语言**：Python（统一技术栈，开发高效）
- **GUI框架**：PySide6（Qt for Python），支持扁平化设计，内置Qt Charts绘图。
- **网络通信**：websockets + asyncio，轻量异步通信。
- **数据库**：SQLite + SQLAlchemy ORM，单文件易部署。
- **量化代理**：初期用规则引擎模拟，后期可集成scikit-learn或ONNX模型。

### 3. 核心模块设计

#### 3.1 服务器端
- **步进控制器**：管理当前时间、模式、步进状态机。负责触发价格更新、策略执行、交易撮合，并协调客户端就绪。
- **价格生成器**：基于几何布朗运动模拟股票价格，生成OHLCV数据。
- **策略引擎**：加载机器人策略（三类），根据当前模式和市场数据生成买卖指令。
- **交易引擎**：处理所有订单（真人手动下单和机器人自动下单），更新账户现金、持仓，记录成交。
- **WebSocket服务器**：处理客户端连接、消息分发（登录、下单、状态同步等），广播步进更新和快进状态。
- **管理员GUI**：股票管理、账户管理、步进控制面板，通过信号槽与后端交互。

#### 3.2 客户端
- **登录界面**：连接服务器，验证身份。
- **主界面**：
  - **图表区域**：根据当前模式显示K线/折线图及技术指标（MACD、KDJ）。
  - **交易面板**：股票代码、数量输入，买入/卖出按钮（步进间隙可用）。
  - **账户信息**：现金、持仓列表。
  - **量化代理开关**：开启后，快进时由服务器自动交易。
  - **通知区域**：显示服务器消息（如快进开始、步进完成）。

### 4. 关键流程

#### 4.1 正常步进流程
1. 管理员选择模式，点击“下一步”。
2. 服务器广播`STEP_WILL_START`，客户端启用交易面板。
3. 真人用户操作后点击“完成”或自动发送就绪信号（若无操作也需确认）。
4. 服务器收集所有真人客户端就绪后，执行一步：
   - 价格生成 → 机器人策略执行 → 交易撮合。
5. 服务器广播`STEP_FINISHED`，包含最新数据，客户端更新界面。

#### 4.2 快进流程
1. 管理员点击“开始快进”。
2. 服务器广播`FASTFORWARD_START`，客户端禁用手动操作，量化代理开关保留。
3. 服务器循环执行步进（价格生成 + 机器人策略 + 托管真人的代理交易），间隔可配置。
4. 管理员点击“停止快进”，服务器广播`FASTFORWARD_STOP`，客户端恢复手动模式并更新状态。

### 5. 数据持久化
- 使用SQLite存储用户、股票、价格历史、持仓、订单。
- 价格历史按步进粒度存储，便于聚合不同周期的K线。

### 6. UI风格
- 扁平化设计，深色背景，简洁控件。
- 图表采用QtCharts，支持缩放、指标叠加。

### 7. 部署
- 服务器端为独立可执行程序（打包为exe），管理员在Windows上运行。
- 客户端同样打包为exe，用户通过IP连接服务器（支持本地局域网联机）。

该方案覆盖所有需求，模块清晰，易于迭代扩展。
