# **股票量化模拟交易系统：详细开发流程文档**

本手册旨在指导开发者从底层数据逻辑到前端界面展示，分阶段完成系统的搭建。

## **阶段 1：数据持久化与模型设计 (核心底座)**

**目标**：建立可靠的数据存储结构，支持所有业务逻辑。

1. **数据库建模 (SQLAlchemy)**：  
   * 定义 User 模型：区分管理员、机器人（存储策略类型）、真人账户。  
   * 定义 Account 与 Position：实现资金与多/空头持仓的关联。  
   * 定义 PriceHistory：存储 OHLCV 数据，支持不同步进周期的数据查询。  
2. **数据初始化脚本**：  
   * 编写初始化脚本，预设 3 类机器人账户并绑定初始策略（散户、机构、游资）。  
   * 预设初始股票清单及其价格生成参数（波动率、初值）。

## **阶段 2：服务端引擎与业务逻辑 (系统大脑)**

**目标**：实现不依赖 UI 的核心模拟算法。

1. **价格生成引擎**：  
   * 实现几何布朗运动 (GBM) 算法。  
   * 开发数据聚合工具，将最小步进数据实时聚合成分钟、小时、天、月 K 线。  
2. **交易与撮合逻辑**：  
   * 编写 TradeManager 类，处理买入、卖出（包括负持仓做空逻辑）。  
   * 计算每一步进后的总资产（现金 \+ 持仓市值）。  
3. **机器人策略引擎**：  
   * 实现策略接口，每类机器人根据当前行情数据（DataFrame 格式）返回买卖信号。  
   * 初期可用规则（如：价格跌 2% 买入）填充，后期接入模型。

## **阶段 3：异步通信协议与步进控制器 (连接中枢)**

**目标**：建立实时互动的“服务端-客户端”通信机制。

1. **WebSocket 协议定义**：  
   * STEP\_INIT：广播初始状态。  
   * STEP\_WAIT：服务端进入等待状态，等待真人操作。  
   * CLIENT\_SUBMIT：客户端提交订单或点击“完成操作”。  
   * STEP\_UPDATE：服务端计算完成，下发新价格和资产信息。  
2. **管理员步进控制器**：  
   * 实现逻辑：只有当所有在线真人账户点击“完成”或管理员“强制下一步”时，才推进步进。  
   * 实现“快进模式”：循环触发步进，跳过等待环节，期间将真人账户设为“托管状态”。

## **4\. 阶段 4：管理员端 (控制中枢) 与 客户端 (交易面板)**

**目标**：完成可视化的桌面应用程序。

1. **服务端 GUI (管理员专用)**：  
   * **账户监视器**：实时列表显示所有账户的资产曲线和当前盈亏。  
   * **步进控制面板**：模式切换（超短线/短线等）及进度条。  
2. **客户端 GUI (真人用户专用)**：  
   * **图表模块**：集成 QtCharts，根据步进模式动态切换折线/K 线。  
   * **交易终端**：下单面板、资产概览、量化代理一键开关。  
   * **UI 优化**：应用扁平化样式表 (QSS)，设计深色/现代感主题。

## **5\. 阶段 5：量化代理集成与压力测试 (系统完善)**

**目标**：提升系统智能度并确保多人联机稳定性。

1. **量化大模型代理**：  
   * 实现“托管模式”：接入本地规则库或远程 API 给出的买卖建议。  
2. **联机性能压测**：  
   * 测试 10+ 真人账户同时在线、50+ 机器人高频交易时的 WebSocket 延迟。  
   * 优化 SQLAlchemy 的写入频率，使用内存缓存减少磁盘 IO。  
3. **打包发布**：  
   * 使用 PyInstaller 将服务端和客户端打包为独立 .exe。

## **开发优先级建议 (Roadmap)**

1. **第一周**：完成阶段 1 和阶段 2 (GBM 价格生成 \+ 交易逻辑)。  
2. **第二周**：完成阶段 3 (基于 qasync 的异步通信架构)。  
3. **第三周**：完成阶段 4 (界面开发，实现“管理员点下一步，客户端数据变动”)。  
4. **第四周**：调优快进功能、图表指标 (MACD/KDJ) 以及最终打包。
